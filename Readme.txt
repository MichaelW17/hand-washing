Cashier_v1是修改了GUI使之能自适应屏幕分辨率的代码；
Cashier_v1_1是用在Ubuntu上的竖屏GUI，控件分布完全改变了，还修改了一些细节功能，比如在每次扫码之后到success/failure之前显示“放在打包区”等。
Cashier_v1_3大量使用setFixedSize，只针对1366x768的竖屏，暂时解决了GUI的问题；此外比较重要的改动包括：能够显示所有已扫码商品;解决了点击"确认付款"之后没有点"是的"而是点击“没有”时按钮显示状态不对的问题
(这种情况下button1会切换为显示"开始结账"的unchecked状态,这是不对的),我的做法是手动通过setChecked
方法将button1固定在checked状态;
Cashier_v2对backend中的状态机做了比较多的改动：添加两种新的处理分支：1）不扫码（不经过扫码区域）直接把东西放在打包区的情况：flag=0的状态下直接跳到flag=2，这样后面就可以正常转移到flag=0，前端发现没扫码，可以报警；（已实现）
                         2）只买一瓶水，不放在扫码区直接拿在手里付款：flag=1且扫码的情况下，直接点击确认付款（暂未实现）
